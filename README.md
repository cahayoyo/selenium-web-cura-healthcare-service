# CURA Healthcare Service Automated Testing Suite

This repository contains a comprehensive automated testing suite for the [CURA Healthcare Service](https://katalon-demo-cura.herokuapp.com/) demo website. The project is built using **Java**, **Selenium WebDriver**, and **TestNG**, following the **Page Object Model (POM)** design pattern to ensure maintainability, scalability, and clarity.

## 🚀 Key Features

- **Page Object Model (POM):** Decouples test logic from UI elements for better maintainability.
- **Comprehensive Test Coverage:** Includes tests for Authentication, Appointment Booking, History, and UI elements.
- **Detailed Reporting:** Utilizes **ExtentReports** for generating rich HTML test execution reports, including screenshots on failure.
- **Logging:** Integrated with **Log4j2** for detailed runtime activity logging.
- **Custom Verifications:** A utility-based approach for soft assertions and standard validations.
- **Configuration Management:** Centralized configuration for URLs, credentials, and test data.

## 🛠️ Tech Stack

- **Language:** Java 17
- **Automation Tool:** Selenium WebDriver (v4.43.0)
- **Framework:** TestNG (v7.12.0)
- **Reporting:** ExtentReports (v5.1.2)
- **Logging:** Log4j2 (v2.25.4)
- **Build Tool:** Maven

## 📁 Project Structure

```
CURAHealthcareServiceTest/
├── src/

│   │   ├── base/              # Base classes (BaseTest)

├── screenshots/               # Screenshots captured on test failure
├── logs/                      # Execution logs generated by Log4j2
├── pom.xml                    # Maven configuration
└── testng.xml                 # TestNG suite configuration
```

## 🏃 Getting Started

### 1. Clone the repository
```bash
git clone https://github.com/your-username/CURAHealthcareServiceTest.git
cd CURAHealthcareServiceTest
```

### 2. Install Dependencies
```bash
mvn clean install
```

### 3. Run Tests
You can run the tests using Maven:
```bash
# Run all tests classes
mvn test

# Run a specific TestNG suite
mvn test -DsuiteXmlFile=testng.xml
```

## 📊 Reporting & Logging

### ExtentReports
Execution reports are automatically generated in the `reports/` folder. They provide a visual summary of test results, execution time, and screenshots for failed test steps.
- **Location:** `reports/ExtentReport_yyyy-MM-dd_HH-mm-ss.html`

### Log4j2 Logging
Detailed execution logs are captured for every test run to assist in debugging.
- **Location:** `logs/application.log`

## 🛠️ Utilities

- **`Config.java`**: Stores environment URLs, credentials, and test data constants.
- **`Helper.java`**: Contains reusable methods for element verification, explicit waits, and URL validation.
- **`Log.java`**: A wrapper for Log4j2 to provide consistent logging across the project.
- **`ExtentReportManager.java`**: Manages the lifecycle of ExtentReports and screenshot capture.
